# Codex App-Server Type Generation

`packages/core/src/generated/codex-app-server/schema/` is generated directly from the local `codex` binary.

## Regenerate (stable API surface)

```bash
pnpm --filter @beanbag/core gen:codex-event-types
```

## Regenerate (stable + experimental API surface)

```bash
pnpm --filter @beanbag/core gen:codex-event-types -- --experimental
```

## What the generator does

1. Runs `codex app-server generate-ts --out <tmp-dir>`.
2. Copies the generated schema files into `schema/`.
3. Rewrites relative imports to include `.js` extensions for NodeNext.
4. Writes `index.ts` as a thin typed wrapper around `ServerNotification` and `EventMsg`.

## Source of truth

- `schema/*.ts`: generated from Codex app-server.
- `index.ts`: hand-maintained wrapper generated by the local script.
- `packages/core/src/types.ts`: derives `ThreadEvent` method/data types from wrapper exports.
